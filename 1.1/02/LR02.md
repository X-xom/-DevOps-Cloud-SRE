##  D2-01 Из лекции 1.1.1

- Linux как универсальный управленец. Операционная система — это базовый слой ПО, который абстрагирует доступ к аппаратному обеспечению. Linux выполняет роль такого менеджера, обеспечивая мультизадачность, распределение памяти и взаимодействие между программами и железом.
- Историческая случайность с открытым кодом. Интересно, что ядро Linux появилось как студенческий проект Линуса Торвальдса, который просто хотел изучить архитектуру нового процессора. Ключевым фактором успеха стала лицензия GPL и объединение с наработками Фонда СПО (проект GNU). Это дало сообществу возможность не только использовать код, но и улучшать его.
- Повсеместность использования. На практике Linux доминирует в серверном сегменте, на суперкомпьютерах и встраиваемых системах. Если вы пользуетесь Android или заходите на любой крупный сайт, вы так или иначе взаимодействуете с Linux-системами. Это фактически стандарт для инфраструктуры.
- Концепция "Всё есть файл". Это фундаментальная абстракция UNIX-подобных систем. Здесь стандартные файлы, директории, а также устройства, сокеты и каналы межпроцессного взаимодействия представлены как файловые объекты. Это унифицирует управление: одними и теми же инструментами можно читать данные с диска, с клавиатуры или из сетевого соединения.
- Модульность и конвейеры. Вместо монолитных приложений, экосистема Linux строится на утилитах, каждая из которых решает одну задачу. Сложные операции выполняются путем объединения этих утилит в конвейеры (pipes), где результат работы одной программы передается на вход другой. Это обеспечивает гибкость и переиспользуемость компонентов.

**Ответы на вопросы «Проверь себя»**

- Ядро — это базовая часть ОС, которая работает с "железом" (процессор, память). Дистрибутив — это ядро + набор программ + менеджер пакетов, собранные в готовую к использованию систему.
- Проект GNU разработал утилиты и окружение, а Linux был создан как ядро. Вместе они сформировали полноценную ОС (GNU/Linux).
- Принцип «Всё есть файл» означает, что к устройству или процессу можно обращаться через файловую систему. Пример: жесткий диск — это файл /dev/sda. Данные с него можно прочитать как из обычного файла.
- Ubuntu (для новичков и десктопа), Astra Linux (для госсектора и задач безопасности РФ), CentOS/Rocky Linux (для серверов, где нужна стабильность).
- Три задачи сисадмина:
    - Следить за работоспособностью серверов (мониторинг).
    - Настраивать безопасность и права доступа.
    - Писать скрипты для автоматизации рутины (бэкапы, обновления).


##  D2-02 Из лекции 1.1.2

 Архитектура Linux (4 слоя):
- Hardware — железо (CPU, RAM, диск).
- Kernel — ядро, управляет доступом к железу.
- System calls / Shell — интерфейс для программ и командная оболочка (bash).
- User space — пользовательские приложения.

 Пользователи:
- Обычный — работает в своем каталоге /home/, ограничен в правах.
- root — суперпользователь, может всё (опасно работать постоянно).
- Группы — объединяют пользователей для общих прав.

- Для удаленного управления используется протокол SSH. Это зашифрованный туннель, по которому мы отправляем команды серверу. Подключаемся так: 
- ssh user@ip_address

**Ответы на вопросы «Проверь себя»**

- Четыре слоя архитектуры: Hardware (железо), Kernel (ядро), System Calls/Shell (системные вызовы/оболочка), User Space (пользовательские приложения).
- Главная функция ядра: Посредничество. Ядро управляет доступом программ к аппаратным ресурсам (процессору, памяти, устройствам ввода-вывода) и распределяет эти ресурсы между запущенными процессами.
- Обычный пользователь vs root: Обычный пользователь работает в своей "песочнице" (в своем домашнем каталоге) и ограничен в правах — он не может сломать системные файлы. Root — это администратор с абсолютными правами, он может делать всё, включая изменение системных файлов и удаление других пользователей.
- Для чего нужны группы: Для упрощения разграничения прав. Можно добавить пользователей в группу и выдать права на файл или каталог для всей группы сразу, а не для каждого пользователя по отдельности. Это удобно для совместной работы над проектами.
- Расшифровка user1@linuxbox:/var/log$:
    - user1 — имя текущего пользователя.
    - linuxbox — имя хоста (сервера), на который он зашел.
    - /var/log — текущий рабочий каталог.
    - $ — признак того, что мы работаем под обычным пользователем (не root).
